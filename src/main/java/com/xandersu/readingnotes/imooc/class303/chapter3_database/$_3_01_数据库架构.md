##索引模块

- 为什么使用索引
    - 避免全表扫描查找数据，提升检索效率
- 什么样的信息能成为索引
    - 主键、唯一键等，具备区分性的数据
- 索引的数据结构
    - 主流B+树，还有哈希和bitmap，mysql不支持bitmap，基于innodb和MyIsam不显式支持hash
- 密集索引和稀疏索引的区别

### 为什么要使用索引

- 快速查询数据

### 什么样的信息能称为索引

主键、唯一键以及普通键等

### 索引的数据结构

- 生成索引，建立二叉查找树进行二分查找
- 生成索引，建立B-Tree结构进行查找
- 生成索引，建立B+-Tree结构进行查找
- 生成索引，建立Hash结构进行查找
